button presses

Up to this point we have relied on using UI elements in Unity (namely the button) to execute commands in our programs. Now we want to turn our attention to using the keyboard as a form of input. Completing this task is actually fairly easy assuming that you have been able to keep up with the content to this point. The one key difference that is of importance is that we will finally be utilizing the function Update().

The functionality of the command to recognize a key press is something that will need to be added to an if statement. Remember, that a decision needs to be made, it is not a simple execution of code.

However, unlike what you might think from a logical perspective, for the use of key presses we generally do not use the else if statement. This is because we do not want to tie a decision about any one key to another key. Each key should be viewed as independent and therefor the if statement should be by itself.

Random Numbers

The last little detail to look at is randomization. To make a random number you first declare a variable and then use Random.Range(min, max); The example below generates a number between 1 and 100.

int myRandomNumber;

myRandomNumber = Random.Range (1, 100);

Functions

Up to this point we have been keeping our code in either the Start() or Update () sections of the code. However, there is a lot more we can do with our code outside of these sections. Both the Start and Update sections are actually something called a function.

A function is a block of code that can be called on to be used whenever we want, but only when we want it. This means that it is not included in our main body of code and is not used constantly throughout the program. This is very valuable because it helps to organize our code, and since all that we need to do is call on the function to use it, this means we do not have to have repeated lines of code in our program.

To create a function we simply do the following:

 

The word public means that it can be called from anywhere in the program. The word void means that the function will have no value; it will only complete a task. And the word(s) myFunction() is just a variable name and can be anything that is a single word.

Of course, making a function is only half of the process, we also need to call our function in order to run the code inside of it. In order to call the function in our example above we simply use the line of code:

myFunction();

With all of this in mind, let's look at a very simple example where we call on a function to print out the words "Hello World".

However, this is a very simple example and only presents the most basic purpose of using functions. In our next example we are going to "send" information to our function that can then be used. This is extremely valuable because this allows us to keep a generic set of code that can take in variables to fit a specific situation.

In this example we are calling the function and sending two pieces of information; in this instance two numbers. We then tell our function what to expect; in this case two numbers. It is important to note that if you setup a function to receive a set number of pieces of information, you must send that same number and type of information when you call it.

The third type type of function is one that "returns" information back to the first function:

In this example, we have a function call doStuff. This function would be called by pushing a button or some other user action. Within the function we have a variable, called thisVariable, that is setup to hold the value that is returned from myFunction. If we then look at my function we see that it returns text. It is also very important to notice that the function is declared as a string (for text and words) rather than void. A void function does not and cannot return a value, and a function can only return the type of value for which it is declared. In this example, myFunction would not be able to return numbers, it can only return text and variables that are declared as string. The information is then returned to our starting function and is used to print.

The final type of function, if you have not already guessed, is a function that both receives and returns information:

In this example, we have a function call doStuff. This function would be called by pushing a button or some other user action. Within the function we have a variable, called thisVariable, that is setup to hold the value that is returned from myFunction. It works just like our last example except that it also sends the specific information to myFunction. So what we see here is just a combination of the previous two examples. Again, it is important to note that myFunction is declared as an integer as that is the value type that it will be returning.
