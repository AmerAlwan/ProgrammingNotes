Selection and functions

The use of selection is extremely important in making any sort of meaningful program. Selection allows the program to make choices based on user input and then run the program accordingly.

The primary way that selection structures are created is by using if statements, else if statements, and else statements. The basic setup for the if statement is as follows:

if (variable > variable)

{

do something

}

You will notice that in this example I use the greater than symbol ( > ) but you can also use other operators:

x < y – less than (true if x is less than y).

x > y – greater than (true if x is greater than y).

x <= y – less than or equal to.

x >= y – greater than or equal to.

= or == - equal to or identical to.

The statement itself the contains a pair of {  }. Anything that you want to happen as a result of the statement being true is contained within these bracket.

An else if statement is set up in the exact same manner as above:

else if (variable > variable)

{

do something

}

The idea of an else if is that there are multiple conditions to be tested and different outputs or results coming from each test.

Finally there is the else statement. This occurs when anything other than the above if/else if statements are true. This can be a great way to catch those unexpected situations where a user enters something or does something that you have not anticipated or do not want. The else statement is presented as follows:

else

{

do something

}

Compound Selection

Up to this point we have looked at selecting only one option at a time. However, there are many situations where we want to account for multiple options. Let's start by looking at the AND option. By using AND we are able to have our selection meet at least two criteria in order to be true.

In our example, our first number must be bigger than our second number AND our first number must be smaller than our third number. We can do the same type of this using OR.

Here the result will be true if either the first number is bigger than the second number OR the first number is smaller than the third number.

These two two examples can have a more realistic application if we consider passwords. For example, both the user name and password have to be correct. Or we could have a situation what the password could be PASSWORD or password.

The final situation, and perhaps the most confusing, is using NOT. NOT is used for any situation where we really do not want something to be true. However, NOT cannot be used by itself, it must be combined with AND NOT or OR NOT.

Exercises

1. Create a program that uses three variables to hold numbers and then three variables to hold the answers to the following equations.

  • num1 + num2 + num3
  • num2 * num3 / num1
  • num3 - num1 + num2

From here, create a series of if statements to find which answer is the largest.

2. Create a program that has four variables; two to hold a user name (one all UPPER CASE, and one all lower case) and two to hold a password (one all UPPER CASE, and one all lower case). Use a single compound if statement to check if the user name and password are correct.

3. Create a program that uses three variables and then two separate compound selection statements using AND, OR, and NOT in both statements. i.e. if var1 > var2 && var3 == var2 || var1 == var3 && ! var2 == var1
