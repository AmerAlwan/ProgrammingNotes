Printing to the Console

Printing the console is an extremely useful tool. Because you cannot always see where a program might have an error, adding in comments to be printed in the console can allow you to narrow down and hopefully pinpoint the error.

Start by going to "GameObject" menu at the top and select "Create Empty". This will add something called GameObject in the hierarchy menu on the left. Technically you can name this element anything that you want as it will never be seen by the user (it is blank). However, there are good naming conventions are important and objects should be named something relevant to their purpose and the name should start with a capital and if there are multiple words, their should be no spaces and each word should be capitalized. For example, ThisIsMyObject.

For the purpose of this program I will name my object MyOutput.

The next thing we should do (if you do not do it right at the start) is save the current scene. A scene is what the player sees and can be a single screen or an entire 3D world. To save the scene simply go to File -> Save Scene As  For this example I will save my scene as MyScene. This should create an icon in your assets panel.

Lastly, we need to create our script. To do this, Right Click in the assets panel which will open a drop down menu. Select Create -> C# Script. This will create a script icon in your assets panel that we will rename ConsoleOutput.

We are now setup and your screen should look like this:

Double-Click the script icon and wait for MonoDevelop to open.

The only thing that we are going to add is the code print ("Hello World"); inside the start method. Now save your script (File -> Save) and return to Unity. Make sure that you console is open (selected) and press play. Did something happen? Probably not. This is because in order for a script to be run it must be connected to an object in Unity. To do this, select your MyOutput object in the hierarchy pane, open the Project tab at the bottom, and then drag your C# Script over to the right into the inspector. variables

A variable is a very important concept in programming. Think of a variable as an empty box. We create that box and then we place a piece of information inside it. You should note here, that a variable can only hold one piece of information. This means that if we want to change the value of a variable, we will lose the original value of that variable.

Generally speaking, we will need to use three types of variables:

  • string: this is to hold words and letters
  • int: this is to hold whole numbers
  • double: this will hold decimal numbers

When we want to print a variable we just need to place it inside the brackets or we "add" it to a string.

We can also complete math inside our print statements.
