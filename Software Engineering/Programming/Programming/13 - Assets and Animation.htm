Assets and Animation

Up to this point we have looked at assets as simple pictures that we have imported, and although this is accurate, there is a lot more out there for us to use. Of course if you happen to be interested in graphic design, you can always use a tool like Blender (free) or Maya (not free) to design your own sprites and animations. But for those of us who not really into that (or just do not have the time), Unity has a massive collection of resources for us to use. Some of these resources are free, while other costs money. For our example we will be using free assets.

Getting Assets

Start by going to the unity website www.unity3d.com and log into your account. From there select the Assets Store tab.

From the assets store you can select a variety of different asset categories. For our example we are going to do a search for "Sprite Sheet".

Then select the price tab and change the value to zero.

Find the "Ninja Sprite Sheet"

When you select the asset, click on the "Add to my Assets" link.

Once it is added, click on "Open in Unity"

You will be asked to open or create a new project; for our example we will make a new project. Your project should then open with the Asset Store tab open and you will see a window with something like this.

Why didn't we just work within Unity to find assets in the asset store? Simplicity. The web interface is easier to use and also works much faster. It is not necessary, just a preference.

Select the small folder icon to open the assets that you have downloaded.

You may need to select the purchased tab, and then select "Download"

The link will then change to "Import".

This will open a window which allows you choose which elements from the assets folder that you want to import. This is a small asset so we will just select "Import". However, many asset packages are very large and you only want to import the parts that you will be using in your project or your final file will be very large and just take more time to load and not actually add anything to the game.

Now, since we are going to be creating multiple animations, we are going to start to organize our elements. First, we will create a new folder called animations in our assets pane.

You probably noticed that the character moves, but none of the animations happen. Let's fix that.

From this point we move into new territory. The first thing that we are going to do is create a variable (in this case I will simply use the letter h) that will keep track of when we use the left or right arrow. To do this we will use the command RawInput which rather than returning the actual input will return a value for our input. In this case it will return a 1 for the right arrow and a -1 for the left arrow.

Now we can create a few if statements to control our character. We will need an if statement for when h = 1, h = -1, and when h = 0. For each situation we will change the value of our walk variable to either true or false (remember our walk variable from when we setup the animations).

Debug your code to make sure everything has been entered properly. Return to Unity and press play.

You should now see your character switch between animations when you press the arrow keys and when you are not pressing anything. However, there is one detail that is kind of annoying; the character is always facing the same direction regardless of which direction it is moving.

Fixing this problem is actually really simple. In Unity check the inspector for an x value in "Scale". This value is going to depend on the size of the object that you are working with so the number in this example is not an absolute value, you will need to go by what you see in Unity. In this example I have not changed the size of the object on the screen so the scale is just 1. You can actually test this by changing the value from positive to negative and you will notice that your character flips and points in the opposite direction.
